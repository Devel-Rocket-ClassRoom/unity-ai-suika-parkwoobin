## 1. 게임 개요

**한 줄 요약**  
물리 기반 과일 낙하 퍼즐 — 같은 과일끼리 충돌하면 합쳐져 더 큰 과일이 되고, 상자를 넘치기 전에 최대한 높은 점수를 획득한다.

**장르**: 2D 물리 퍼즐 / 캐주얼  
**플랫폼**: PC (Unity 2022 이상, URP 2D)  
**조작**: 마우스 클릭(또는 터치) — 수평 위치 지정 후 낙하

### 코어 루프

```
과일 낙하 → 착지·정착 → 동종 충돌 → 머지(업그레이드) → 점수 획득
                                        ↓
                             상자 밖으로 넘치면 게임 오버
```

## 2. 핵심 메커닉

### 2-1. 떨어뜨리기 (Drop)
- 화면 상단에 드롭 라인이 존재하며, 플레이어는 수평 위치를 조절해 과일을 낙하시킨다.
- 현재 드롭할 과일과 **다음 과일 미리보기** 1개가 항상 표시된다.
- 낙하 중인 과일은 충돌 전까지 좌우 이동 가능 (선택 구현).
- 드롭 가능 과일 범위: 레벨 1~5 (Cherry ~ Persimmon). 6 이상은 머지로만 생성.

### 2-2. 물리·충돌 (Physics)
- Unity Physics2D 사용 (`Rigidbody2D` + `CircleCollider2D`).
- 과일마다 물리 반경이 다름 (크기 비율표 참고).
- Restitution(탄성): 낮게 설정하여 통통 튀지 않고 자연스럽게 쌓임.
- 마찰: 과일끼리 미끄러지지 않도록 적당한 마찰 부여.

### 2-3. 머지 (Merge)
- **동일 레벨** 과일 두 개가 충돌하면 즉시 머지 발생.
- 머지 위치: 두 과일의 중간 지점.
- 결과물: 다음 레벨 과일 1개 생성.
- 연쇄 머지 가능 (머지된 과일이 또 같은 과일과 충돌).
- 최고 레벨(수박, Lv.11)은 더 이상 머지되지 않는다.

### 2-4. 게임 오버
- 과일이 상자 상단 경계선(위험선) 위로 **일정 시간(약 3초) 이상** 머물면 게임 오버.
- 낙하 직후 일시적으로 선을 넘는 것은 허용 (판정 유예).
- 게임 오버 시 최종 점수와 베스트 스코어 표시 후 리트라이 선택.

## 3. 과일 단계 (Fruit Levels)

| Lv | 이름 | 영문 | 반지름 (상대값) | 머지 점수 | 드롭 가능 |
|----|------|------|:--------------:|:---------:|:---------:|
| 1  | 체리 | Cherry | 1.0 | 1 | ✓ |
| 2  | 딸기 | Strawberry | 1.4 | 3 | ✓ |
| 3  | 포도 | Grape | 1.8 | 6 | ✓ |
| 4  | 데코폰 | Dekopon | 2.4 | 10 | ✓ |
| 5  | 감 | Persimmon | 3.0 | 15 | ✓ |
| 6  | 사과 | Apple | 3.8 | 21 | ✗ |
| 7  | 배 | Pear | 4.6 | 28 | ✗ |
| 8  | 복숭아 | Peach | 5.5 | 36 | ✗ |
| 9  | 파인애플 | Pineapple | 6.5 | 45 | ✗ |
| 10 | 멜론 | Melon | 7.8 | 55 | ✗ |
| 11 | 수박 | Watermelon | 9.5 | — | ✗ |

> 반지름 단위: 체리를 1로 했을 때의 상대 비율. Unity 단위 환산은 구현 단계에서 결정.  
> 머지 점수: 해당 레벨 과일이 생성될 때 획득. Lv.11은 머지 불가이므로 점수 없음.

## 4. 점수 · 승패 규칙

### 점수 계산
- 머지가 발생할 때마다 위 표의 **머지 점수**를 현재 점수에 누적.
- 연쇄 머지는 순서대로 각각 점수 획득.
- 보너스 없음 (단순 누적 방식) — 확장 시 콤보 보너스 고려 가능.

### 승리 조건
- 없음. 게임 오버 없이 최대한 오래, 높은 점수를 노리는 엔드리스 방식.

### 패배 조건
- 과일이 상자 위험선 위에 지속적으로 넘침 → 게임 오버.

### 기록
- 현재 세션 점수 (Current Score).
- 베스트 스코어 — `PlayerPrefs`로 로컬 저장.

## 5. 화면 · UI 구성

```
┌─────────────────────────────┐
│  [SCORE: 0000]  [BEST: 0000]│  ← 상단 HUD
│                             │
│      [다음 과일 미리보기]   │  ← 드롭 라인 오른쪽 상단 (소형)
│           ↓ 현재 과일       │
│   ─ ─ ─ ─위험선─ ─ ─ ─    │  ← 빨간 점선
│  │                       │  │
│  │     [상자 내부]        │  │
│  │    (과일들 쌓임)       │  │
│  │_______________________|  │
└─────────────────────────────┘
```

| 요소 | 위치 | 설명 |
|------|------|------|
| 현재 점수 | 상단 좌측 | 실시간 업데이트 |
| 베스트 스코어 | 상단 우측 | 로컬 저장값 |
| 다음 과일 미리보기 | 상단 우측 소형 박스 | 다음에 나올 과일 1개 표시 |
| 위험선 | 상자 상단 경계 | 빨간 점선, 게임 오버 기준 |
| 드롭 가이드선 | 현재 과일에서 수직으로 | 착지 위치 예측 (선택) |
| 게임오버 오버레이 | 화면 중앙 | 최종 점수, 베스트, 리트라이 버튼 |

## 6. 톤 & 분위기

### 컬러 팔레트
- 배경: 크림 화이트 (`#FFF8E7`) — 밝고 포근한 느낌
- 상자 테두리: 연한 나무 갈색 (`#C8A870`)
- 위험선: 산호색 빨강 (`#FF6B6B`) 점선
- UI 폰트: 둥글둥글한 산세리프 (예: M PLUS Rounded, Noto Sans KR)

### 아트 방향
- **2D 원형 스프라이트** 기반. 각 과일은 귀엽고 단순화된 일러스트 스타일.
- 머지 시 작은 파티클(빛 번짐 + 별 이펙트) 재생.
- 배경은 단색 또는 옅은 그라데이션으로 과일에 시선 집중.

### BGM · SFX 방향
- BGM: 가볍고 밝은 J-Pop 스타일 루프. BPM 120~130.
- SFX:
  - 낙하: 부드러운 "뽁" 소리
  - 머지: 맑은 "팅" + 레벨에 따라 음이 높아짐
  - 게임오버: 짧은 하강 음악
